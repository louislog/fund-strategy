## 基金策略系统开发计划

版本：v1.0  
更新时间：2026-04-28  
适用仓库：`fund-strategy`

---

### 1. 背景与现状（审阅结论）

- **系统定位**：纯前端静态 Web 应用，通过 JSONP 拉取基金/指数历史数据，在前端完成回测与可视化。
- **当前回测核心**：`src/utils/fund-stragegy/index.ts` 内 `InvestmentStrategy` + `InvestDateSnapshot`，完成**单基金**日级快照推演（买入/卖出/工资入账/分红处理/收益指标）。
- **当前策略实现方式**：止盈/补仓等触发条件主要位于 `src/pages/index.tsx` 的 `onEachDay` 回调中（表单参数 + if 组合判断），属于**页面驱动策略**，存在以下问题：
  - **可复用性弱**：策略逻辑难以在对比页/组合页复用与组合；
  - **可扩展性弱**：新增策略需改页面；策略之间优先级、互斥难管理；
  - **可解释性不足**：交易触发原因难以被结构化记录并在 UI 复盘。
- **数据能力**：
  - 基金净值：`src/utils/fund-stragegy/fetch-fund-data.ts#getFundData`
  - 指数：`getIndexFundData` + `calcMACD` + `txnByMacd`（MACD 百分位与交易点标记）
- **页面能力**：
  - 基础回测页：`src/pages/index.tsx`
  - 策略对比页：`src/pages/compare/*`（复用单基金回测，多套“保存条件”并行跑出曲线）
  - 策略保存：`src/pages/components/saved-search.tsx`（localStorage）

---

### 2. 目标与范围

#### 2.1 业务目标（对应任务）

1) **丰富当前策略模块功能（通用策略设计）**  
将策略拆为三类并模块化：**投资策略**、**补仓策略**、**止盈策略**。支持策略组合、优先级、可解释交易原因。

2) **新增投资多只基金、基金持仓、持仓回测功能**  
引入**组合（Portfolio）**概念，支持多基金持仓与现金统一记账，输出组合曲线、持仓结构、交易流水与复盘视图。

3) **优化 Web 页面交互逻辑**  
提升“配置→回测→看结果→复盘→保存/对比”的闭环体验，提供更明确的加载/错误态与交易解释。

#### 2.2 范围边界（避免需求漂移）

- **本阶段不做**（可列入后续 Roadmap）：
  - 真实交易接入、实时行情、账户同步、后端服务与数据库落库
  - 税费、申赎确认延迟、赎回到账日等更真实的交易规则（可后续扩展）
  - 高级统计（因子归因、风险模型、优化器）与大规模参数寻优

### 3. 总体设计方案（推荐）

核心原则：**策略（产出信号）与执行（更新持仓）解耦**，并让“单基金回测”和“组合回测”共用一套动作/流水模型。

#### 3.1 领域分层

- **策略层（Signal/Policy）**：只根据“当日快照 + 指标 + 配置”产出 `Action[]`
- **执行层（Execution）**：接收 `Action[]`，执行买卖、扣费、更新现金/份额/成本，产出 `Transaction[]`
- **引擎层（Engine）**：迭代交易日，计算/读取指标，调用策略层与执行层，生成 `Snapshot[]`

#### 3.2 建议新增目录（落点建议）

- `src/utils/fund-stragegy/types.ts`：统一类型（策略配置/动作/流水/快照）
- `src/utils/fund-stragegy/actions/`：动作定义与撮合执行（Buy/Sell/Noop、金额/份额）
- `src/utils/fund-stragegy/strategies/`：策略实现（定投、MACD 补仓、分批止盈、回撤止盈等）
- `src/utils/fund-stragegy/engine/`：回测循环与调度（单基金/组合）
- `src/utils/fund-stragegy/portfolio/`：组合模型（Portfolio、Position、PortfolioSnapshot）

> 迁移目标：`src/pages/index.tsx` 不再承载复杂的交易 if 判断，只负责组装策略配置与展示结果。

### 4. 数据结构草案（用于对齐实现）

> 以下为“计划级草案”，最终以实现为准，但建议在 Sprint 1 确立并全项目统一使用。

#### 4.1 Action（策略输出）

- `Action`
  - `type`: `BUY | SELL`
  - `fundId`: string
  - `date`: string
  - `amount?`: number（以金额下单）
  - `portion?`: number（以份额下单）
  - `reason`: string（可展示给用户的触发原因）
  - `strategyId`: string（策略标识，便于复盘/过滤）
  - `tags?`: string[]（如 `fixed`, `take_profit`, `add_position`）

#### 4.2 Transaction（执行结果）

- `Transaction`
  - `date`, `fundId`, `side`
  - `price`（成交净值）
  - `portion`, `amount`（含/不含费率需在字段约定中统一）
  - `fee`
  - `cashDelta`（现金变化）
  - `strategyId`, `reason`, `tags`

#### 4.3 Snapshot（快照输出）

- 单基金：沿用/扩展 `InvestDateSnapshot`，并新增关联的 `transactions` 或在引擎层统一汇总
- 组合：
  - `PortfolioSnapshot`: `date`, `total`, `cash`, `positionsValue`, `profit`, `profitRate`, `drawdown?`, `weightsByFund?`
  - `PositionSnapshot`: `fundId`, `portion`, `cost`, `value`, `profit`, `weight`

### 5. 里程碑与交付计划（可按人力合并/拆分）

建议按 4 个 Sprint 交付（每个 Sprint 都可上线/可验收）。

#### Sprint 1：策略模块重构（单基金不变，结构先定）

- **目标**：把“页面 if 交易逻辑”迁入策略模块；建立统一 Action/Transaction 模型；保证回测结果一致。
- **主要工作**
  - 从 `src/pages/index.tsx` 中抽离止盈/补仓条件判断，封装为策略模块（至少覆盖当前：MACD 补仓 + 指数/仓位/收益率/新高止盈）
  - 引入 `Action`/`Transaction`，每次交易都记录 `reason/strategyId/tags`
  - 引擎执行顺序与冲突规则（最小可用版）：同日多动作按优先级执行（如先卖后买）
- **验收标准**
  - 同一组表单参数运行，重构前后曲线一致（允许极小舍入误差）
  - `index.tsx` 不再包含复杂 if 的交易逻辑
  - 结果数据里可以拿到“交易流水 + 原因”

#### Sprint 2：策略能力丰富（通用策略体系补齐）

- **目标**：补齐“投资/补仓/止盈”三类策略的通用实现，并支持组合使用。
- **交付内容**
  - **投资策略**：
    - 定投统一抽象（交易日/每周/每月）与可配置化
    - 可选：初始建仓规则（一次性/分批）
  - **补仓策略（至少 2 类）**：
    - 回撤补仓：从阶段高点回撤达到阈值分批买入
    - 仓位补仓：低于目标仓位补到目标（或区间仓位控制）
    - 保留：MACD 补仓（现有能力迁移后的版本）
  - **止盈策略（至少 2 类）**：
    - 分批止盈：达到收益率阈值，按比例/固定金额卖出
    - 回撤止盈：盈利后回撤达到阈值减仓/清仓
- **验收标准**
  - 策略可开关、可组合；同日冲突可解释（流水中体现）
  - 每笔交易都能追溯到策略与原因

#### Sprint 3：多基金组合与持仓回测（Portfolio v1）

- **目标**：从单基金扩展到多基金组合，输出组合层快照与持仓结构。
- **交付内容**
  - 新增组合配置：基金列表、初始现金、每月现金流入（工资）、资金分配方式（等权/固定比例/目标权重）
  - 组合回测引擎：输出 `PortfolioSnapshot[]`、`PositionSnapshot[]`、`transactions[]`
  - 组合结果页（基础版）：组合总资产曲线、现金/仓位曲线、持仓占比、交易流水
- **验收标准**
  - 3~5 只基金、3 年区间回测在可接受时间内完成（前端不卡死）
  - 组合中任意基金移除/禁用，组合曲线随之变化且逻辑自洽

#### Sprint 4：Web 交互优化（体验与复盘闭环）

- **目标**：让用户能“配置→运行→看懂→复盘→保存/对比”闭环完成，不依赖控制台。
- **交付内容（建议优先级）**
  - 统一加载态/错误态（基金无效、区间超出运营期、接口失败、数据为空）
  - 策略编辑体验：策略模块卡片化（投资/补仓/止盈），可折叠、可开关、一键恢复默认
  - 复盘增强：图表标注 buy/sell 点；交易流水表支持按基金/策略/标签筛选
  - 保存与对比升级：保存“策略方案/组合方案”而不只是表单字段；对比页支持组合维度
- **验收标准**
  - 新用户可在 3 分钟内完成一次回测并理解每笔交易原因

### 6. 关键风险与应对

- **第三方数据源结构变化**（JSONP 字段可能变更）
  - 应对：封装数据适配层；失败时提示并允许重试；缓存最近成功数据
- **多基金时间对齐**（净值缺失导致组合跳变）
  - 应对：统一“取最近有效净值”的规则；必要时对缺失日做标注
- **前端性能压力**（长区间、多基金、多策略）
  - 应对：计算分片/节流；必要时引入 Web Worker（作为后续优化点）

### 8. 验收用例（建议最少集）

- **一致性用例**：使用一组固定表单参数，重构前后单基金曲线一致
- **止盈/补仓可解释性**：任意一次交易都能在流水中看到 `strategyId + reason`
- **组合用例**：2~3 只基金等权定投，组合总资产曲线可生成；禁用其中一只基金后曲线改变
- **异常用例**：输入不存在基金/超出基金运营期/接口失败时，页面有明确提示且不崩溃
